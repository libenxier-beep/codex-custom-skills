# Source Quality Rubric

Use this before promoting source material into durable knowledge.

## Quality Dimensions

| Dimension | High signal | Low signal |
| --- | --- | --- |
| Provenance | Author, organization, URL, date, context are clear. | Anonymous, clipped, or unverifiable. |
| Evidence | Shows data, examples, code, cases, or mechanisms. | Only asserts conclusions. |
| Transferability | Mechanism can be adapted across contexts. | Works only because of hidden local conditions. |
| Specificity | Changes future behavior. | Generic advice already known. |
| Counterexamples | Names limits, tradeoffs, or failure modes. | Claims universal applicability. |
| Freshness | Time-sensitive claims are current or checked. | Likely stale and unverified. |
| Originality | Adds a new mechanism, lens, or practice. | Repackages common advice. |

## Rating

- `high`: strong provenance, mechanism, evidence, and durable applicability.
- `medium`: useful but partial, context-bound, or needs more validation.
- `low`: interesting but speculative, vague, stale, or weakly sourced.
- `reject`: do not store as durable knowledge; optionally mention in the report.

## Conflict Handling

When sources disagree:

- Preserve the disagreement instead of blending it away.
- Identify which assumptions differ.
- Store as an open question if evidence is insufficient.
- Promote a conditional principle only when the condition is clear.

## Copyright And Privacy

- Store links, source cards, short excerpts, and derived knowledge.
- Avoid copying full protected works into memory.
- For private user material, keep source references useful but minimal.
- When in doubt, store the abstraction and enough provenance to find the original later.
